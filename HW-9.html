  2. What state \(\ell\) satisfies \(\ell \preceq x\) for all \(x \in \{0, 1\}^n\)?

Consider the following Gibbs step for the Ising model, which is monotonic when \(\beta > 0\).

step_local_gibbs <- function(x_n, i, u, beta) {
  n_0 = sum(x_n == 0)
  n_1 = sum(x_n == 1)
  return(as.integer(u < (exp(beta * n_1) / (exp(beta * n_1) + exp(beta * n_0)))))
}

Then the following code uses CFTP to generate draws exactly from the Ising model.

cftp_local_gibbs <- function(n = 1, g, beta = 1) {
  steps <- 2^n
  num_nodes <- g %>% activate(nodes) %>% as_tibble() %>% nrow()
  i <- floor(num_nodes * runif(steps)) + 1
  u <- runif(steps)
  x <- rep(0, num_nodes)
  w <- rep(1, num_nodes)
  for (t in 1:steps) {
    neigh <- neighbors(g, i[t])
    x_n <- x[neigh]
    x[i[t]] <- step_local_gibbs(x_n, i[t], u[t], beta)
    w_n <- w[neigh]
    w[i[t]] <- step_local_gibbs(w_n, i[t], u[t], beta)
  }
  if (sum(x) == sum(w) | n > 10) {
    return(x)
  }
  else {
    y <- cftp_local_gibbs(n + 1, g, beta)
    for (t in 1:steps) {
      y_n <- y[neighbors(g, i[t])]
      y[i[t]] <- step_local_gibbs(y_n, i[t], u[t], beta)
    }
    return(y)
  }
}

Then the following calculates \(h(x)\) for a configuration on a graph.

h <- function(x, g) {
  from <- g %>% activate(edges) %>% pull(from)
  to <- g %>% activate(edges) %>% pull(to)
  return(sum(x[from] == x[to]))
}

bonferroni_ar_p5 <- function() { 
  a <- FALSE
  while (!a) {
    #choose a point
    i <- floor(5 * runif(1)) + 1
    
    #determine location of ith point
    r <- sqrt(runif(1.2))
    theta <- 2 * pi * runif(1)
    points <- matrix(rep(0, 10), ncol = 2) # set up points matrix 
    
    for (j in 1:5) {
      if (j == i)
        points[j,] <- c(2.1 + r * cos(theta), 3.5 + r * sin(theta))
      else
        points[j,] <- 10 * runif(2) }
    
    u <- runif(1)

    m <- sum(sapply(1:5, function(j) (sqrt(sum((points[j,] - c(2.1, 3.5))^2)) <= 1.2)))
    a <- (u < 1 / m) 
  }
return(points) 
}

Consider the following probabilistic recursive algorithm for generating a geometric random variable with parameter \(p\).

rgeo_onedraw <- function(p) {
  u <- runif(1)
  if (u < p) 
    return(1)
  else
    return(1 + rgeo_onedraw(p))
}

Show that this algorithm is locally correct for output \(G\) such that \(\mathbb{P}(G = i) = (1 - p)^{i - 1} p \mathbb{I}(i \in \{1, 2, 3, \ldots\})\).

Suppose \(G\) is output thergeo_onedraw algorithm. Let \(Y\) be the output of the recursive call. We know \(\mathbb{P}(Y = i) = (1 - p)^{i - 1} p\). Prove original.

Let \(G\) be the output of the call to the algorithm. For any \(i\): \[P(G=i)=P(G=X, X=i)+P(G=Y, Y=i)\] \[P(G=i)=P(G=X | X=i) \cdot P(X=i) +P(G=Y | Y=i) \cdot P(Y=i)\] \[P(G=i)= p \cdot P(X=i) + (1-p) \cdot P(Y=i)\] \[P(G=i) - p \cdot P(X=i) = (1-p) \cdot P(Y=i)\] Assuming that \(P(G=i)\) and \(P(X=i)\) are the same, because they are both based on the rgeo_onedraw algorithm: \[(1-p) \cdot P(G=i) = (1-p) \cdot P(Y=i)\] \[P(G=i) = P(Y=i) = (1 - p)^{i - 1} p\] Because we supposed that \(Y\) was an oracle. In short, we assumed the recursive call was true and used it to prove that the original algorithm was locally correct.
